## Overview

In this tutorial, we will design a **schematic-only** ESP32 module circuit using tscircuit. This design follows the style taught in hardware academy courses and covers all the essential supporting circuitry for the bare **ESP32-D0WDQ6** SoC.

We will use JLCPCB-stocked components where possible, and focus purely on the logical schematic — no PCB layout is applied.

---

## 📋 Components Checklist

Our module schematic will consist of:
1. **ESP32-D0WDQ6 SoC (U1)** — Dual-core 240 MHz Wi-Fi + BT chip
2. **ME6211C33 LDO (U2)** — 3.3 V/500 mA regulator (JLCPCB basic)
3. **CH340C USB-Serial (U3)** — USB-to-UART with integrated oscillator
4. **40 MHz Crystal (Y1)** — External clock for the ESP32
5. **Decoupling capacitors** — Per-pin power stabilisation
6. **EN / BOOT buttons** — Manual reset and bootloader entry
7. **Chip antenna + pi-network** — 2.4 GHz RF front-end

### 2. 3.3 V LDO Regulator

The **ME6211C33** is a compact SOT-23-5 LDO that converts 5 V (USB) to 3.3 V for the ESP32. Add bulk and decoupling capacitors on both input and output rails.

```tsx title="Esp32Module.tsx (power section)"

        name="U2"
```

### 3. 40 MHz Crystal Oscillator

The ESP32 needs an external 40 MHz crystal on its XTAL_P and XTAL_N pins. Use two 10 pF load capacitors as specified in the datasheet.

```tsx title="Esp32Module.tsx (crystal section)"
      {/* ── 40 MHz Crystal ── */}
      <chip
        name="Y1"
        schPortArrangement={{
          leftSide:  { pins: ["pin1"], direction: "top-to-bottom" },
          rightSide: { pins: ["pin2"], direction: "top-to-bottom" },
        }}
        pinLabels={{ pin1: "XIN", pin2: "XOUT" }}
        schX={4}
        schY={-9}
      />
      <capacitor name="C10" capacitance="10pF" schX={2} schY={-10} schRotation="90deg" />
      <capacitor name="C11" capacitance="10pF" schX={6} schY={-10} schRotation="90deg" />

      <trace from=".U1 .XTAL_P" to=".Y1 .XIN" />
      <trace from=".U1 .XTAL_N" to=".Y1 .XOUT" />
      <trace from=".Y1 .XIN"    to=".C10 .pos" />
      <trace from=".Y1 .XOUT"   to=".C11 .pos" />
      <trace from=".C10 .neg"   to="net.GND" />
      <trace from=".C11 .neg"   to="net.GND" />
```

### 4. CH340C USB-to-Serial Bridge

The **CH340C** has a built-in 12 MHz oscillator — no external crystal is needed. Its TXD/RXD cross-connect to the ESP32 UART0 pins. Two 100 nF coupling capacitors on DTR and RTS implement the **auto-reset** circuit used by esptool to enter the bootloader without manually pressing buttons.

```tsx title="Esp32Module.tsx (USB-serial section)"
      {/* ── CH340C USB-to-UART ── */}
      <chip
        name="U3"
        manufacturerPartNumber="CH340C"
        schPortArrangement={{
          leftSide: {
            pins: ["pin1", "pin2", "pin3", "pin4", "pin7", "pin8"],
            direction: "top-to-bottom",
          },
          rightSide: {
            pins: ["pin5", "pin6", "pin9", "pin10", "pin11", "pin16"],
            direction: "top-to-bottom",
          },
        }}
        pinLabels={{
          pin1:  "GND",  pin2:  "TXD",  pin3: "RXD",
          pin4:  "V3",   pin5:  "UD+",  pin6: "UD-",
          pin7:  "XI",   pin8:  "XO",
          pin9:  "CTS",  pin10: "DSR",  pin11: "RI",
          pin16: "VCC",
        }}
        schX={-6}
        schY={-6}
        schPinSpacing={0.4}
      />
      {/* VCC decoupling */}
      <capacitor name="C12" capacitance="100nF" schX={-4}  schY={-9} schRotation="90deg" />
      {/* Auto-reset: DTR → C13 → EN, RTS → C14 → IO0 */}
      <capacitor name="C13" capacitance="100nF" schX={-8}  schY={-10} schRotation="0deg" />
      <capacitor name="C14" capacitance="100nF" schX={-10} schY={-10} schRotation="0deg" />

      <trace from=".U3 .VCC"    to="net.5V" />
      <trace from=".U3 .GND"    to="net.GND" />
      <trace from=".U3 .V3"     to="net.3V3" />
      <trace from=".U3 .TXD"    to="net.ESP_RX" />
      <trace from=".U3 .RXD"    to="net.ESP_TX" />
      <trace from=".U1 .RXD0"   to="net.ESP_RX" />
      <trace from=".U1 .TXD0"   to="net.ESP_TX" />
      <trace from=".C12 .pos"   to="net.3V3" />
      <trace from=".C12 .neg"   to="net.GND" />
      {/* Auto-reset coupling */}
      <trace from=".C13 .pin1"  to=".U3 .DTR" />
      <trace from=".C13 .pin2"  to=".U1 .EN" />
      <trace from=".C14 .pin1"  to=".U3 .RTS" />
      <trace from=".C14 .pin2"  to=".U1 .IO0" />
```

## 💡 Key Design Notes

1. **Crystal load capacitors** — The ESP32 datasheet specifies 10 pF for the 40 MHz oscillator. Wrong values cause Wi-Fi connection instability and failed calibration.

2. **Auto-reset circuit** — C13 and C14 implement the standard DTR/RTS toggle used by `esptool`. Without them, you must hold BOOT and press RESET manually every time you flash firmware.

3. **ME6211C33 vs AMS1117** — The ME6211C33 is a JLCPCB "basic" part (always in stock, no placement fee). The AMS1117-3.3 also works but is a "preferred" part with a small extra cost.

4. **CH340C vs CH340G** — The CH340C has an internal 12 MHz oscillator so it saves the external crystal and two capacitors compared to the CH340G variant.

5. **Antenna keep-out** — On a real PCB, place the chip antenna at a board edge with a copper-free keep-out zone underneath to avoid detuning the antenna.
